# Log Management and Rotation

The Age Gate as a Service application generates logs via Pino (structured logging). To prevent unbounded disk usage, log rotation and retention must be configured both in development (Docker Compose) and production (K3s).

## Docker Compose (Local Development)

In `docker-compose.test.yml`, we have added logging limits for the TimescaleDB and Redis containers:

```yaml
logging:
  driver: "json-file"
  options:
    max-size: "10m"
    max-file: "3"
```

This limits each container log file to 10 MB and keeps up to 3 rotated files (total 30 MB).

For the `agegate-verifier` application (when run locally via `node server.js`), logs are written to stdout/stderr. You can redirect them to a file and use `logrotate`:

```bash
node server.js >> app.log 2>&1
```

## Kubernetes (K3s Production)

In a K3s cluster, you can configure log rotation via **kubelet flags**. Add the following to `/etc/rancher/k3s/config.yaml`:

### Option 1: Kubelet flags (recommended)

```yaml
kubelet-arg:
  - "container-log-max-size=10Mi"
  - "container-log-max-files=5"
```

Then restart K3s:

```bash
sudo systemctl restart k3s
```

These limits apply to all containers in the cluster. Each container will keep at most 5 log files of 10 MB each (total 50 MB). The logs are stored in `/var/log/containers/`.

### Option 2: Sidecar container with logrotate

If you need per‑pod configuration or cannot modify node settings, add a sidecar container to the `agegate-verifier` deployment. This sidecar rotates the log file generated by the main application.

Example snippet for `deployment.yaml`:

```yaml
containers:
  - name: logrotate
    image: bitnami/logrotate:latest
    volumeMounts:
      - name: logs
        mountPath: /var/log/app
    command:
      - /bin/sh
      - -c
      - |
        cat > /etc/logrotate.conf <<EOF
        /var/log/app/app.log {
          size 10M
          rotate 5
          compress
          missingok
          notifempty
        }
        EOF
        while true; do
          logrotate -s /tmp/logrotate.status /etc/logrotate.conf
          sleep 3600
        done
```

Then mount an `emptyDir` volume shared between the main container and the sidecar:

```yaml
volumes:
  - name: logs
    emptyDir: {}
```

In the main container, configure Pino to write to `/var/log/app/app.log` (instead of stdout). Alternatively, redirect the application’s stdout to that file by modifying the container command.

### Option 3: Node‑level logrotate (advanced)

You can also configure `logrotate` on the host OS to rotate logs in `/var/log/containers/`. This is node‑specific and not recommended for multi‑node clusters.

**Example configuration** (`/etc/logrotate.d/k3s-containers`):

```bash
/var/log/containers/*.log {
    size 10M
    rotate 5
    compress
    missingok
    notifempty
}
```

Apply with:
```bash
sudo logrotate -f /etc/logrotate.d/k3s-containers
```

(The built‑in kubelet flags (Option 1) remain the recommended approach for production.)

## Monitoring disk usage

Use Prometheus and Grafana to monitor disk usage of the nodes and alert when log storage exceeds thresholds. The metric `node_filesystem_avail_bytes` can be used to track free space.
