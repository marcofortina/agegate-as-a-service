# Security Measures

## CSRF Protection

The admin dashboard uses **CSRF tokens** to protect state-changing endpoints (register, revoke, rotate, update rate limit, update description). The token is generated by the `csurf` middleware and embedded in the HTML page as a meta tag and a global `window.csrfToken` variable.

All `fetch` requests from the admin dashboard include the token in the `CSRF-Token` header. Requests without a valid token are rejected with a `403` status.

## Authentication

Admin access is protected by HTTP Basic Authentication. Credentials are passed via environment variables (`ADMIN_USER`, `ADMIN_PASS`). The session is maintained using an httpOnly cookie after the first login.

## Double Anonymity

Client IP addresses are never logged or stored; only an irreversible SHA256 hash with a daily rotating salt is kept. Rate limiting uses the hashed IP.

## HTTPS

In production, the service should be deployed behind a reverse proxy with HTTPS enabled. Security headers (Helmet) and CORS are configured conditionally based on the `PUBLIC_URL` protocol.
